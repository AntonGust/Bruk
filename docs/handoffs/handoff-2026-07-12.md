## The Gateway design in 30 seconds (ADR-0010 is authoritative)

One plain-Envoy pod per cluster, rendered by the Airon operator like any other workload, owning
the OpenAI-compatible contract:

- **Full contract shape day one:** single public base URL, model picked from the request body
  (Lua filter → routing header), `GET /v1/models` catalog. The URL shape is the
  hard-to-change artifact.
- **Config lives on `BrukTenant.spec.gateway`** (optional, additive to v1alpha1): `hostname`
  (via `${GATEWAY_HOSTNAME}` postBuild.substitute — no box coordinates in the public repo),
  `tls.secretRef`, `apiKeysSecretRef`. **Every Ready InferenceService is auto-routable** by
  `servedModelName` — routing table and catalog are the same join; smoke model included
  (feature: free e2e health check).
- **Auth:** `Authorization: Bearer <key>`; named Secret entries (overlap-then-retire rotation);
  validated by an **ext_authz sidecar = second entrypoint of the bruk-operator image**
  (`gateway-authz`), constant-time compare, file-watch reload. The operator never touches key
  values — ADR-0008 posture intact. Auth before routing; unknown model → OpenAI-style
  `model_not_found`.
- **Catalog + routes via filesystem xDS** (operator renders RDS/CDS + catalog into the
  ConfigMap; Envoy hot-reloads — zero pod restarts on model changes).
- **TLS:** standard `kubernetes.io/tls` Secret consumed via filesystem SDS; produced by
  cert-manager (new Flux HelmRelease) + Let's Encrypt **DNS-01 via Cloudflare** (airon.ai NS is
  Cloudflare); zone-scoped API token Secret readable by cert-manager only; port 80 never opens.
- **Exposure:** `hostNetwork: true` on :443 — a named single-node-era exception with a full
  hardening list; **named successor** = normal pod behind a LoadBalancer Service, per-cluster VIP
  announced to the ToR (Cilium BGP or BlueField-3 LB), decided when the fabric routing design
  exists. Migration is a render change + DNS update by design.
- **Fleet framing:** 500–1000 customers = 500–1000 single-tenant clusters stamping the pattern;
  Envoy is never the bottleneck. Real fleet pressure points: DNS automation + **LE's ~50 new
  certs/week/domain onboarding cap** (request an increase before mass onboarding).
- **Rate limiting:** Envoy local token buckets now (engine-queue protection + cheap-reject
  unauthenticated); per-key quotas deferred to the metering increment.
- **Pinned break-the-product settings:** route timeouts must fit multi-minute SSE streams
  (Envoy default 15 s kills them); buffer **request** bodies only — responses stream unbuffered.

Read `docs/explanations/gateway-explained.md` for the guided version (simple first, then every
component).

## Next steps — Gateway implementation (3.2 discipline: TDD, CI, golden tests)

**Human prerequisites (Anton, blocking step 4/5 only):**

1. Create the DNS record under airon.ai pointing at the box (name = future
   `${GATEWAY_HOSTNAME}`).
2. Mint a Cloudflare API token scoped to the airon.ai zone (DNS edit + zone read) and create the
   Secret for cert-manager out-of-band (same class as `hf-token`).

**Engineering order:**

1. `BrukTenant.spec.gateway` API fields + CRD regen (additive, optional).
2. Render package: Envoy Deployment (hostNetwork + hardening list) + ConfigMap (bootstrap,
   filesystem-xDS RDS/CDS, catalog direct_response, Lua model-routing filter, local rate limits,
   ext_authz wiring) + golden tests beside the workload goldens.
3. `gateway-authz` entrypoint in the operator binary: ext_authz server, constant-time compare,
   Secret-file watch. Security-critical — TDD it.
4. cert-manager HelmRelease + ClusterIssuer (Cloudflare DNS-01) through the M3 adoption pattern.
5. Go-live on the box: `${GATEWAY_HOSTNAME}` substitution variable, DNS record, verify with an
   unmodified OpenAI SDK against the public URL (chat completion + streaming + `/v1/models` +
   401 on bad key).

## Standing gotchas (unchanged — pointers only)

- Changing `${INITDATA_B64}` rolls **both** CC pods → the 24B re-downloads ~90 GB.
- Never mix CC and non-CC GPUs on one node; verify blast radius before restarting
  k3s/containerd/GPU-operator pieces.
- One pod per trusted-store PVC (LUKS double-format corrupts).
- Helm-values edits under `install/helm-values/` ARE live cluster changes (Flux upgrades on
  merge).
